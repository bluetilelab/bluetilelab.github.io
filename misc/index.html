    Documentation Rules

    1. 비아냥대지 않는다

      • 본 페이지 개설 목적은 코드 품질 및 생산성 향상을 위해 규칙을 정하고 이를 따르자는 것이다
      • 모르는 것을 비난하려는 태도로 작성하지 않는다

    2. 객관적인 사실에 근거하여 작성한다

      • 근거없이 지레짐작하지 않는다.
      • 잘못한 이해에 기반한 코드가 계속 쌓이면 프로젝트가 걷잡을 수 없이 무너진다
      • 공식문서로도 모르겠으면 공신력있는 외부 근거를 찾아 참조한다
        • 참조들은 문서 하단에 링크를 명시한다
      • 본인 개발 환경에서 측정한 결과를 참조할 수 있다

    3. 코드 예제 및 그림 등을 첨부한다

      • 아무리 간단한 개념이라도 단어 선택, 어조에 따라 해석 차이가 발생할 수 있다.
      • 개념 파악을 위해 글을 읽어야 한다는 것 자체가 피로감을 느끼게 한다
      • 따라서, 글 외에 이해를 돕는 코드 예제, 그림, 테이블 등의 보조 자료를 적극적으로 활용한다
